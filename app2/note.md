# 高雄附近房價地圖 App — 凍結規格（MVP / 本機 Demo）

## 一句話目標

打開 App → 確認在高雄市 → 2D 地圖顯示自己位置與附近成交點（依屋齡 + 每坪價著色）→ 點擊看摘要 → 細節連結先佔位。資料為實價登錄彙整，大約就好，風格簡約可愛。

---

## 產品決策（已凍結）

| 項目 | 決定 |
|---|---|
| 範圍 | 僅高雄市 |
| 資料來源 | 內政部實價登錄（成交價），非公告地價 |
| 匯入方式 | 每月 / 每季 CSV → 本機 Postgres + PostGIS |
| 「眼前這棟」 | 接受約略：使用者位置 **100m** 內最近一筆成交 |
| 附近查詢 | 半徑 **500m**，最多回傳 **20** 筆（依距離最近） |
| 地圖 | **2D only**；iOS 用 `react-native-maps`（Apple 地圖） |
| 著色 | **坪價 + 屋齡一起**：填色 = 坪價；框線 = 屋齡 |
| 細節連結 | Demo 先佔位（coming soon / TBD） |
| 資料時間窗 | 近 **2 年內**成交（本季 CSV 先當 demo 資料） |
| 無定位權限 | **不能使用** App |
| 不在高雄 | 顯示提示後阻擋使用 |
| 免責聲明 | **必備**（非估價、非即時、公開資料彙整） |
| 目標 | **本機先跑通的 demo**（不上架、不架雲端） |
| Geocode | **免費 NLSC**（國土測繪中心），結果快取進 DB |

### 不做（MVP Out of Scope）

- 全台、3D / Three.js 房子、AR
- 帳號、收藏、降價通知
- 付費地圖（Mapbox）/ 付費 geocode（Google）
- 保證門牌＝眼前實體建築的法律級精度
- 完整房仲交易平台

## 使用者流程

1. 啟動 → 請求定位權限；拒絕則無法使用  
2. 判斷座標是否在高雄市行政區內  
   - 否 → 「本 App 目前僅支援高雄市」並停止  
   - 是 → 進入地圖  
3. 地圖：自己的位置 + 500m 內 ≤20 個著色點 + 圖例 + 免責  
4. 點擊某點 → 底部摘要：屋齡、每坪價、成交日、建物型態、距離  
5. 「看細節」→ 外部連結佔位（TBD）  
6. 另可顯示「眼前最近一筆」（100m 內）；若無資料則空狀態文案

## 資料檔

- 範例：`app2/115-1-K.csv`（高雄市，民國 115 年第 1 季實價登錄）  
- 約 8255 筆有效列；**第 2 行為英文表頭，匯入時須跳過**  
- CSV **沒有經緯度**，只有門牌 → 必須 geocode  

### 建議保留欄位

| CSV 欄位 | 用途 |
|---|---|
| `交易標的種類` | 篩選建物相關交易 |
| `建物型態` | 顯示（華廈 / 大樓 / 透天…） |
| `房地總價` | 詳情 |
| `單價元平方公尺` | 轉成元/坪（核心） |
| `交易日期` | 民國年月日 → 西元；新鮮度 |
| `代表建號屋齡` | 著色；可能缺值 |
| `建築完成年月` | 屋齡缺值時推算 |
| `代表地號或門牌` | 顯示 + NLSC geocode key |
| `行政區` | 區級資訊 / 後備 |
| `備註欄` | 過濾特殊交易 |
| `主建物面積`、`幾房`、`幾廳`、`幾衛`、`電梯` | 詳情加分（可選） |

### 可忽略欄位（MVP）

土地/車位數量細節、都市與非都市分區、地段、代表地號（此檔多為空）、車位類別與樓層、建材、層次、附屬/陽台面積、有無隔間等。

## 資料處理 Pipeline

1. **讀檔**：UTF-8；跳過英文表頭列  
2. **篩選交易**  
   - 保留：`房地(土地+建物)`、`房地(土地+建物)+車位`、`建物`  
   - 排除：純 `土地`、純 `車位`  
3. **必要欄位**  
   - `單價元平方公尺` 非空且 > 0  
   - `代表地號或門牌` 含「號」（門牌，非純地號）  
4. **衍生欄位**  
   - `unit_price_ping` = `單價元平方公尺 × 3.30579`  
   - `tx_date`：民國 `YYYMMDD` → 西元（年 + 1911）  
   - `building_age`：優先 `代表建號屋齡`；否則用 `建築完成年月` 推算  
5. **品質過濾（建議）**  
   - `備註欄` 含「親友 / 員工 / 共有人 / 特殊關係 / 瑕疵 / 急買急賣」等可排除  
   - 極端單價：以分位數或合理區間剔除（避免歪色階）  
6. **同門牌去重（地圖用）**  
   - 同一地址多筆 → 保留最新 `交易日期`（或中位坪價）；避免同棟堆疊多點  
7. **Geocode（NLSC）**  
   - 以門牌呼叫免費 NLSC 地址定位 / 模糊檢索  
   - 成功 → 寫入 `lat,lng`；失敗 → 不上圖  
   - **結果必須快取**（同址不重複打 API）  
8. **寫入 DB** → 建立 PostGIS 空間索引  

### Geocode：免費 NLSC

- 提供者：內政部國土測繪中心（NLSC）國土測繪圖資服務雲  
- 說明：https://maps.nlsc.gov.tw/S09SOA  
- 用途：門牌 → `lat/lng`（批次在 import 時做，不在使用者每次開 App 時打）  
- 注意：部分 API（如模糊檢索）可能需**免費申請介接**後使用；demo 前確認申請狀態與呼叫方式  
- 規範：限速、錯誤重試、快取表；失敗列記 log，不阻斷整批匯入

## DB Schema（建議最小集）

```text
transactions
  id
  address_text          -- 代表地號或門牌
  district              -- 行政區
  building_type         -- 建物型態
  total_price           -- 房地總價（元）
  unit_price_m2         -- 單價元/平方公尺
  unit_price_ping       -- 單價元/坪（衍生）
  building_age_years    -- 屋齡（衍生後）
  tx_date               -- 成交日（西元 date）
  rooms / halls / baths -- 可選
  has_elevator          -- 可選
  note_flag             -- 是否特殊交易
  lat, lng
  geom                  -- PostGIS Point(4326)
  source_file           -- 如 115-1-K.csv
  created_at, updated_at

indexes: GIST(geom), (district), (tx_date), (address_text)
geocode_cache(address_text UNIQUE, lat, lng, provider, updated_at)
```

高雄市邊界：可用行政區 polygon 做 `in-kaohsiung` 判斷（或簡化 bbox + 區名單，demo 可先簡化）。

## 著色規則（建議預設）

- **填色（坪價）**：依高雄建物成交分位數分桶（例：低 / 中 / 高；可用該季 p10 / p50 / p90 當錨點）  
- **框線（屋齡）**：例 `0–10` / `11–30` / `31+`  
- 地圖旁固定圖例；色數少、可讀優先

## API（FastAPI，本機）

| Method | Path | 說明 |
|---|---|---|
| GET | `/health` | 健康檢查 |
| GET | `/geo/in-kaohsiung?lat=&lng=` | 是否在高雄 |
| GET | `/properties/nearby?lat=&lng=&radius_m=500&limit=20` | 附近點（輕量：id, lat, lng, age, ping_price, color buckets, distance） |
| GET | `/properties/nearest?lat=&lng=&max_m=100` | 眼前最近一筆（可無資料） |
| GET | `/properties/{id}` | 詳情（連結欄位先 null / placeholder） |
